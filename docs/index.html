Task Bridge API

Single backend for the mobile app, web UI, and automation agents. Create epics, manage workflow stages, post comments, run the worker queue, and link library documents through this API.

Base URL: http://localhost:3000 Format: JSON Auth: X-Api-Key

Default port

3000 — override with TASK_BRIDGE_PORT

API key

BACKEND_API_KEY — sent on every protected request

Public endpoints

GET /health and GET /connect.json require no auth

Authentication

All protected routes expect an X-Api-Key header. The value must match the server BACKEND_API_KEY. Missing or invalid keys return 401 Unauthorized.

EXAMPLE Any protected endpoint 
curl -s http://localhost:3000/projects \
  -H "X-Api-Key: dev-key"

The mobile app discovers the server endpoint and API key via GET /connect.json.

Error format

Most errors return a plain JSON object.

Connect

Discovery endpoint for mobile pairing. Returns the host where the API is reachable and the API key the app should use. Whatever URL you deploy or expose the server on is what clients should connect to — there is no fixed tunnel provider.

GET /connect.json Server connection manifest (no auth)
Response 200
→ ConnectManifest

Epics & Tasks

Epics (parentId: null) are workflow roots. Subtasks attach to an epic or another task. Detail responses include isEpic, subtasks, comments, and stage.

POST /epics Create epic (workflow auto-spawned)
FieldDescription
projectIdTarget project
title / textAt least one required; first line of text becomes the title
descriptionOptional long-form text
 
curl -X POST http://localhost:3000/epics \
  -H "X-Api-Key: dev-key" \
  -H "Content-Type: application/json" \
  -d '{"projectId":"task-bridge","title":"New feature","description":"Details..."}'
Response 201
→ CreatedTask
POST /tasks Add task under an epic
FieldTypeRequired
parentIdnumberyes
titlestringyes
descriptionstringno
stageIdstringno
Response 201
→ CreatedTask
GET /tasks List all bridge tasks (raw)
Response 200
→ TaskListResponse
GET /tasks/:id Task or epic detail
Response 200
→ TaskDetail
PATCH /tasks/:id Update comment or description

Epic: description and/or comment

Subtask: comment only

{
  "comment": { "text": "Review complete", "by": "mobile" },
  "description": "Updated epic description"
}
Response 200
→ TaskDetail

Values: todo · in_progress · done. Epics return 400.
